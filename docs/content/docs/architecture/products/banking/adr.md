## **Architectural Decision Record (ADR)**

**Название задачи:** Концептуальная архитектура открытия депозитов для MVP<br></br>
**Дата:** 01.08.2025

## **Функциональные требования**

| №  | Код   | Действующие лица или системы                 | Система       | Use Case                       | Описание                                                                                                |
|----|-------|----------------------------------------------|---------------|--------------------------------|---------------------------------------------------------------------------------------------------------|
| 1  | FR-1  | Новый клиент                                 | Сайт          | Просмотр депозитов на сайте    | Новый клиент заходит на сайт банка и видит список доступных депозитов с актуальными ставками.           |
| 2  | FR-2  | Новый клиент                                 | Сайт          | Подача заявки на сайте         | Новый клиент оставляет заявку на открытие депозита, указав номер телефона и Ф.И.О.                      |
| 3  | FR-3  | Менеджер кол-центра                          | Кол-центр     | Обработка заявки               | Менеджер анализирует заявку клиента, связывается с ним по телефону и предлагает индивидуальные условия. |
| 4  | FR-4  | Клиент интернет-банка                        | Интернет-банк | Просмотр персональных ставок   | Зарегистрированный клиент в интернет-банке видит актуальные и персонализированные ставки по депозитам.  |
| 5  | FR-5  | Клиент интернет-банка                        | Интернет-банк | Подача заявки в интернет-банке | Клиент подаёт заявку на депозит, указывая счёт и сумму, и подтверждает операцию СМС-кодом.              |
| 6  | FR-6  | Сотрудник бэк-офиса депозитов (в рамках MVP) | АБС           | Согласование ставки            | Сотрудник бэк-офиса проверяет и подтверждает предложенную клиенту ставку по заявке на депозит.          |
| 7  | FR-7  | Сотрудник бэк-офиса кредитов (в рамках MVP)  | АБС           | Обработка ставок               | Сотрудник бэк-офиса кредитов работает с XLS-файлами для актуализации ставок по депозитам и кредитам.    |
| 8  | FR-8  | Клиент интернет-банка                        | СМС-шлюз      | Получение СМС-уведомлений      | Клиент получает СМС о подтверждении ставки и открытии депозита.                                         |
| 9  | FR-9  | Менеджер бэк-офиса (в рамках MVP)            | СМС-шлюз / ИБ | Уведомление клиента            | Менеджер отправляет клиенту уведомление о статусе заявки на депозит через СМС или пуш-уведомление.      |
| 10 | FR-10 | Сотрудник бэк-офиса депозитов/кредитов       | АБС           | Администрирование ставок       | Сотрудник изменяет и добавляет ставки депозитов и кредитов через интерфейс в АБС.                       |
| 11 | FR-11 | Менеджер кол-центра                          | Кол-центр     | Использование CRM              | Менеджер использует систему кол-центра для управления контактами клиентов и выполнения звонков.         |
| 12 | FR-12 | Система интернет-банка и АБС                 | АБС / ИБ      | Обработка данных депозита      | Системы взаимодействуют для передачи данных о заявке на депозит без прямого API-взаимодействия.         |

## Нефункциональные требования

| №  | Требование                                                                                                                                  |
|----|---------------------------------------------------------------------------------------------------------------------------------------------|
| 1  | Система интернет-банка должна быть доступна 24/7 и обеспечивать 99,9% времени доступности.                                                  |
| 2  | Время загрузки страниц интернет-банка не должно превышать одной секунды.                                                                    |
| 3  | Время отклика API для всех систем не должно превышать 500 миллисекунд.                                                                      |
| 4  | Система должна поддерживать не менее 10 000 одновременных пользователей интернет-банка без деградации производительности.                   |
| 5  | Данные, передаваемые через интернет-банк и сайт, должны быть защищены с помощью механизма шифрования трафика.                               |
| 6  | В случае сбоев в ЦОД интернет-банк должен переключаться на резервный ЦОД и оставаться доступным.                                            |
| 7  | Системы должны быть устойчивы к отказам и поддерживать механизм восстановления данных (backup/restore).                                     |
| 8  | Системы должны обеспечивать равномерное горизонтальное масштабирование и распределение запросов между серверами и ЦОД.                      |
| 9  | Доступ к системам с конфиденциальной информацией должен быть ограничен через многофакторную аутентификацию.                                 |
| 10 | Системы должны соответствовать требованиям регуляторов, включая нормативы защиты персональных данных (например, GDPR, ФЗ-152).              |
| 11 | Интерфейс всех систем должен соответствовать утверждённой дизайн-системе банка.                                                             |
| 12 | Визуальные элементы интерфейса должны быть адаптированы для работы как на настольных, так и на мобильных устройствах.                       |
| 13 | Для упрощения диагностики ошибок и анализа производительности должен быть предусмотрен централизованный механизм логирования и мониторинга. |
| 14 | Система должна обеспечивать минимальное время простоя при переходе на новые версии или внесении изменений.                                  |

## Недостатки, ограничения, риски

| Категория          | Элемент                                                                                                  |
|--------------------|----------------------------------------------------------------------------------------------------------|
| **Недостатки**     |                                                                                                          |
| Инфраструктура     | Интернет-банк и АБС работают в рамках монолитной архитектуры, что усложняет масштабирование и доработки. |
| Поддержка          | Обновление ядра интернет-банка зависит от подрядчика, что замедляет реализацию изменений.                |
| Совместимость      | АБС поддерживает только вертикальное масштабирование из-за особенностей БД.                              |
| Производительность | Интернет-банк медленно обрабатывает справочные данные, что ухудшает пользовательский опыт.               |
| Ставки депозитов   | Данные о ставках хранятся в XLS-файлах - сложности обработки и анализа.                                  |
| Интеграции         | Текущая версия интернет-банка несовместима с Kafka.                                                      |
|                    |                                                                                                          |
| **Ограничения**    |                                                                                                          |
| Технологические    | Новые технологии должны быть совместимы с MS SQL и Oracle и соответствовать текущей экспертизе банка.    |
| Безопасность       | Необходимость шифрования трафика для передачи данных в интернет-банке и на сайте.                        |
| Масштабируемость   | Только интернет-банк может горизонтально масштабироваться; АБС ограничена вертикальным масштабированием. |
| Регуляторика       | Системы должны соответствовать требованиям защиты персональных данных (ФЗ-152).                          |
| Финансовые         | Привлечение подрядчика для доработок может быть затратным.                                               |
|                    |                                                                                                          |
| **Риски**          |                                                                                                          |
| Доступность        | Сбой в ЦОД без быстрого переключения на резервный может привести к недоступности интернет-банка.         |
| Производительность | Увеличение количества пользователей может перегрузить базу данных АБС, что приведёт к сбоям.             |
| Безопасность       | Уязвимости в системах шифрования могут привести к утечке конфиденциальной информации.                    |
| Зависимость        | Полная поддержка системы кол-центра осуществляется подрядчиком, что создаёт риск зависимости от него.    |
| Технологические    | Низкий уровень экспертизы сотрудников в новых технологиях может замедлить внедрение изменений.           |
